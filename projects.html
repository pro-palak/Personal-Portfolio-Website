TaskAid

A Task Management Tool for ADHD individuals

TaskAid is a web-based tool designed specifically for individuals with ADHD to improve task management and productivity. The platform focuses on simplicity, clarity, and reducing distractions, allowing users to stay focused and organized.

Key features include a one-task-at-a-time mode, visual aids for task prioritization, and a clean, minimalist design to reduce cognitive load. The app also includes customizable reminders and a progress tracker to motivate users as they complete tasks. Currently, TaskAid is in development and has not been deployed yet. You can check out the code and contribute to the project on GitHub.

TaskAid is built with a modern, scalable tech stack that ensures it runs smoothly across all devices: Frontend: HTML, CSS, and JavaScript (for dynamic, interactive elements) Backend: PHP and XAMPP (for server-side functionality and database management) Database: MySQL (to store and manage user data) Design Tools: Photoshop and Figma (for UI/UX design) Hosting: Currently being prepared for deployment

Parcel Management System

For my assignment in Software Architecture, I developed a Depot Parcel Processing System, which simulates depot workers processing customer parcels. The application follows a three-tier architecture and utilizes the Model-View-Controller (MVC) pattern to structure the user interface.

The system consists of several key classes, such as Parcel, Customer, QueueOfCustomers, and Log, all implemented using appropriate design patterns. The Singleton pattern was applied in the Log class to ensure that only one instance of the log exists throughout the program, recording events like customer queue entries and parcel processing. The system also features fee calculations, parcel sorting, and the management of queues, with the Worker class handling the core processing logic.

A GUI was developed using Java Swing to display a real-time view of the current queue of customers, parcels to be processed, and details of the parcel being processed. As I built the GUI iteratively, I followed the MVC pattern to separate the application logic, user interface, and data handling, ensuring maintainability and scalability. I scored 96/100 for this assignment, demonstrating my ability to implement and integrate architectural patterns effectively while focusing on user experience.
