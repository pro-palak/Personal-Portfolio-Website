Parcel Management System

For my assignment in Software Architecture, I designed and implemented a Depot Parcel Processing System to simulate the workflow of depot workers handling customer parcels. The application adheres to a three-tier architecture (presentation, logic, data) and employs the Model-View-Controller (MVC) pattern to decouple the user interface from business logic, ensuring modularity and ease of maintenance.

The system's core functionality revolves around key classes like Parcel, Customer, QueueOfCustomers, and Log, each implemented with deliberate design patterns. For instance:

  • Singleton Pattern: Applied to the Log class to guarantee a single instance for tracking events (e.g., queue entries, parcel processing) system-wide.
  • Strategy Pattern: Used for dynamic fee calculations based on parcel type (e.g., standard, express).
  • Observer Pattern: Enabled real-time GUI updates by notifying the view layer of state changes in queues or parcels.

The Java Swing GUI provides an interactive dashboard displaying:

  • Real-time customer queues with priority handling.
  • Parcel processing status (e.g., weight, fee, destination).
  • A logging panel for auditing system events.

Outcome: The project earned 96/100, with praise for its robust architecture, adherence to design patterns, and intuitive UI. It underscored my ability to translate theoretical concepts (e.g., SOLID principles, layered design) into functional, maintainable software.

ActivitySense: Human Activity Classification

ActivitySense is a machine learning project that classifies human movement patterns (e.g., walking, running, resting) from sensor data using Principal Component Analysis (PCA) for dimensionality reduction and Support Vector Machines (SVM) for classification. Developed in Python with Jupyter Notebook, the system processes real-world accelerometer/gyroscope datasets to demonstrate how raw sensor signals can be transformed into interpretable activity insights.

Technical Implementation:

  • Data Pipeline: Leveraged pandas for cleaning and normalizing noisy sensor data, handling missing values, and segmenting time-series signals into sliding windows for feature extraction.
  • Dimensionality Reduction: Applied PCA to condense 30+ raw features (e.g., mean, variance, FFT coefficients) into 3 principal components, retaining 95% variance while reducing computational overhead.
  • Model Tuning: Optimized SVM hyperparameters (C, gamma, kernel selection) via grid search, achieving 92% accuracy on test data. Compared performance against baseline models (Random Forest, Logistic Regression).
  • Visualization: Used matplotlib and seaborn to plot PCA loadings, confusion matrices, and decision boundaries, emphasizing interpretability for non-technical stakeholders. Highlighted trade-offs between model complexity (e.g., RBF kernel vs. linear SVM) and real-time deployment constraints.

Outcome: The project not only achieved high classification accuracy but also served as a case study in end-to-end ML workflow design‚Äîfrom data wrangling to actionable insights. Techniques used here are directly applicable to wearable tech, healthcare monitoring, and IoT edge devices.
